# OMF Vitrine Project Memory

## Project Overview
OMF Vitrine is a one-page website that showcases the OMF (Open MagicDraw Framework) features and capabilities. The website is built using Next.js, React, TypeScript, and Tailwind CSS.

## Key Components
- Navbar: Navigation component with mobile responsiveness
- Section: Base component for all main sections
- Feature-specific sections: WhatsOMF, Core, Architecture, FeatureItems, Library, FeatureLibrary, TestFramework
- CodeSnippet/CodeSnippetV2: Components for displaying code examples
- PlaceholderMedia: Component for displaying GIFs and images
- Footer: Footer component with links and information

## Tech Stack
- React
- Next.js
- TypeScript
- Tailwind CSS
- Framer Motion for animations
- Shadcn UI for UI components
- Firebase for hosting

## Mobile Responsiveness
We've improved the mobile responsiveness of the website by:
1. Adding proper viewport meta tags
2. Using responsive class names with Tailwind (sm, md, lg prefixes)
3. Creating mobile-friendly navigation
4. Adjusting font sizes for smaller screens
5. Restructuring layouts for mobile views
6. Ensuring sections take up full width on mobile
7. Using more appropriate font sizes for mobile
8. Adapting the CodeSnippet component UI for mobile screens

### Section Component Mobile Adjustments:
- Changed title font sizes from arbitrary large values to more responsive sizes
- Added `w-full` class to ensure sections take full width on mobile
- Adjusted spacing and alignment for better mobile display
- Used more appropriate font sizes that scale well across different devices

### CodeSnippet Mobile Adjustments:
- Added dynamic max height calculation based on screen size
- Reduced padding on mobile screens
- Made the copy button more touch-friendly
- Adjusted line number display for better mobile experience
- Added responsive text sizing

## Deployment Process
The website is deployed using Firebase Hosting. The deployment process involves:
1. Building the Next.js project
2. Deploying the built files to Firebase
3. Viewing the deployed website at https://omf-website-48649.web.app

## Scripts
- `deploy-mobile.ps1`: PowerShell script for deploying with mobile improvements
- `start-dev.ps1`: PowerShell script for starting development server with port fallback
- `check-mobile.ps1`: PowerShell script for testing mobile responsiveness

## Troubleshooting
- PowerShell commands: Use semicolons (;) instead of ampersands (&&) for command chaining
- Port conflicts: The development scripts now handle port conflicts by trying alternative ports
- Build errors: Fixed linter warnings and errors
- Responsive testing: Test on various device sizes using the mobile testing guide

## Notes on Development
- Mobile-first approach: Start with mobile design and enhance for larger screens
- Responsive images: Use appropriate sizing and lazy loading
- Performance: Keep bundle size small, optimize images and animations
- Accessibility: Ensure proper contrast, focus states, and semantic HTML
